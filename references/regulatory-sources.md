# NIS2 Regulatory Sources Catalog

Quick-reference catalog of official EU and German sources. Consult when you need to cite authoritative guidance or direct the user to official resources.

## EU-Level Sources

| Source | Date | Binding? | Relevance |
|--------|------|----------|-----------|
| **Directive (EU) 2022/2555** | Dec 2022 | Yes (via transposition) | The NIS2 Directive itself — the baseline for all assessments |
| **CIR (EU) 2024/2690** | Oct 2024 | Yes (directly) | Technical requirements for digital infrastructure/provider entities (DNS, TLD, cloud, data centres, CDN, MSP/MSSP, online marketplaces, search engines, social networks, trust services). Flag when assessed entity falls into these sectors — they face granular requirements beyond Art. 21 |
| **ENISA Technical Implementation Guidance** | June 2025 | No | Practical advice, evidence examples, and mappings to ISO 27001 for the 13 CIR Annex areas. Available at enisa.europa.eu |
| **Commission NIS2/DORA Guidelines** | Sept 2023 | No | Article 4 lex specialis guidance — financial entities under DORA are excluded from NIS2 Art. 21 and Art. 23 |
| **Commission NIS2 amendment proposal** | Jan 2026 | No (proposed) | Certification-based compliance, small midcap category, scope clarifications (electricity >1MW, healthcare, hydrogen, chemicals), expanded scope (digital wallets, submarine infrastructure). Expected adoption late 2026/2027. Mention for strategic awareness only |

## German BSI Sources

| Source | Relevance |
|--------|-----------|
| **BSI #nis2know Infopakete** | Detailed guidance per measure + Meldepflicht, Geschäftsleitungsschulung, DORA/NIS2, sector-specific (bsi.bund.de) |
| **BSI #nis2know Roadmap** | 6-phase implementation: (1) Analyse & Grundsatzklärung → (2) Organisation & Verantwortung → (3) Ist-Zustand & Risikobewertung → (4) Planung von Ressourcen → (5) Umsetzung Kernmaßnahmen → (6) Verstetigung & Verbesserung |
| **BSI-Standards 200-1/200-2/200-3** | ISMS methodology and risk management — recommended by BSI as orientation for NIS2 compliance |
| **BSI RUN methodology** | 5-level maturity (Geplant → Kontinuierlich verbessert) — originally for KRITIS Nachweisprüfung, applicable universally |
| **BSI Betroffenheitsprüfung** | Free scope self-assessment tool (betroffenheitspruefung-nis-2.bsi.de). Non-binding, no evidentiary value — useful first step only |
| **BSI CyberRisikoCheck (CRC)** | Basic IT security check. BSI explicitly states this does NOT achieve NIS2 conformity — several § 30 requirements are not covered |
| **BSI Geschäftsleitungsschulung Handreichung** | Management body training guidance |
| **BSI TR-03191 (CSAF)** | Common Security Advisory Framework for automated vulnerability disclosure — useful for supply chain measure |
